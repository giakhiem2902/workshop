## AWS Meet Up – 11/07/2026

**Theme:** Cloud Architecture, AWS Security, Monitoring & AWS Cloud Practitioner

### Purpose of the Event

- Help students access practical knowledge on designing system architecture on the AWS platform.
- Share AI-based solutions for Web application security and the DevSecOps process.
- Provide knowledge on Service Level Agreement (SLA), Monitoring, and managing Cloud system operations.
- Guide the study path and preparation for the AWS Certified Cloud Practitioner (CLF-C02) certification.
- Create opportunities to network and learn from experts and the AWS Student Builder community.

### Speaker List

- Thinh Nguyen – DevOps / DevSecOps / Cloud Engineer, Styl Solutions
- Nguyen Huynh Son – Infrastructure Support Engineer, Endava
- Ngo Le Tan Huy – Speaker, AWS Certified Cloud Practitioner

### Highlights

**1. Cloud Architect Competition Final**

The program opened with the final round of the Cloud Architect competition, featuring the two best teams: Team Ngu Dai Hiep and Team KLKAT.

Both teams presented system architecture solutions built on AWS, analyzed how to choose the right Cloud services for the problem's requirements, and presented their deployment approach, scalability, availability, security, and cost optimization.

Besides the presentations, the teams answered many questions from the judges regarding architecture design thinking, how to handle real-world scenarios, and the ability to optimize performance and operations on AWS.

After evaluation, Team KLKAT was crowned the champion of the Cloud Architect competition.

This session helped me learn more about the AWS-standard architecture design process, how to choose services suitable for each problem, and presentation and defense skills for technical solutions in front of an expert panel.

**2. Securing Your Web Apps With AWS Security Agent**

Speaker Thinh Nguyen introduced AWS Security Agent, a Generative AI-based solution that helps automate the security testing process for Web applications. Instead of only performing manual tests, the AI Agent can automatically analyze system architecture, review source code, and perform penetration testing to detect security vulnerabilities during software development.

Through this session, I gained a clearer understanding of:

- The limitations of traditional security testing methods, such as being time-consuming, costly, and dependent on security experts.
- AWS Security Agent, built on Amazon Bedrock, capable of planning and automatically executing security tasks.
- The Design Review, Code Review, and Automated Penetration Testing process across the entire software development lifecycle.
- The ability to evaluate architecture against standards such as PCI DSS, NIST CSF, and the AWS Well-Architected Framework.
- Integration with GitHub and GitLab to automatically check Pull Requests, detect vulnerabilities, and suggest appropriate patches.
- The ability to simulate multi-step attack chains to verify vulnerabilities rather than just issuing warnings.
- The limitations of AI Security Agents for systems using multi-factor authentication (MFA), biometric authentication, or complex business logic.

This session helped me better understand the trend of applying AI in DevSecOps, while also recognizing the important role of automation in improving the quality and security level of Web applications.

**3. SLA and Monitoring – From SLA to Monitoring: What Really Matters**

Speaker Nguyen Huynh Son shared insights on the role of Service Level Agreements (SLA) and Monitoring in ensuring the service quality of modern Cloud systems. Through many real-world examples, the speaker emphasized that a system with stable infrastructure doesn't necessarily mean users have a good experience. It's essential to track the entire service usage journey rather than only monitoring CPU, Memory, or Network.

Through the presentation, I gained a clearer understanding of:

- The Service Level Agreement (SLA) concept and its role in ensuring service quality.
- Monitoring as an important part of risk management, helping detect incidents before they affect users.
- The Monitoring Pyramid model, covering Cloud Provider, Infrastructure, Application, Business Metrics, and Customer Experience.
- The message "Healthy Infrastructure ≠ Healthy User Experience," emphasizing that infrastructure can run normally while users still encounter errors.
- How to use Amazon CloudWatch, CloudWatch Alarm, and Amazon SNS to monitor systems, set up alerts, and support incident response.
- The importance of tracking business metrics such as Login Success Rate, successful transaction rate, and user experience rather than only focusing on resource metrics.

Through this session, I understood the mindset of building a user-centric Monitoring system, which improves operational capability and ensures service quality.

**4. Inside the Exam – AWS Certified Cloud Practitioner**

Speaker Ngo Le Tan Huy gave an overview of the AWS Certified Cloud Practitioner (CLF-C02) certification, and shared a study roadmap and exam-preparation experience for those just starting to learn about AWS Cloud.

Through this session, I learned:

- The exam structure: 65 questions in 90 minutes, covering four main knowledge areas: Cloud Concepts, Security and Compliance, Cloud Technology and Services, and Billing, Pricing and Support.
- The six key benefits of AWS Cloud along with the design principles of the AWS Well-Architected Framework.
- The Shared Responsibility Model, which distinguishes security responsibilities between AWS and the customer.
- Basic knowledge of IAM, Security Groups, Network ACL, AWS WAF, AWS Shield, and AWS Artifact.
- The role of core services such as Amazon EC2, AWS Lambda, Amazon S3, Amazon RDS, Amazon DynamoDB, Amazon VPC, and Amazon Route 53.
- The pricing models of Amazon EC2, AWS Cost Explorer, AWS Budgets, and AWS support plans.
- Study experience, practice exams, hands-on practice with the AWS Free Tier, and exam tips for achieving a good result.

This session gave me an overview of the AWS ecosystem and a clearer direction for studying and preparing for the AWS Certified Cloud Practitioner certification.

## Week 7 (01/06 – 07/06)

### Objectives
- Research AWS services suitable for the Budget Tracker project.
- Analyze the business requirements for a personal expense management system.
- Build a deployment approach for the system on AWS Serverless.

### Activities

**01/06**
- Studied the Serverless architecture on AWS.
- Reviewed Amazon API Gateway, AWS Lambda, and Amazon DynamoDB.

**02/06**
- Analyzed the requirements of the Budget Tracker system.
- Defined the core features: transaction management, budget management, category management, AI expense analysis, and budget-overrun alerts.

**03/06**
- Researched the AWS services to be used: Amazon Route 53, Amazon CloudFront, AWS WAF, Amazon S3, Amazon API Gateway, Amazon Cognito, AWS Lambda, Amazon DynamoDB, Amazon Bedrock, Amazon SNS, Amazon CloudWatch, Amazon SQS.

**04/06**
- Built a Serverless deployment architecture.
- Evaluated scalability and availability.

**05/06**
- Designed the user processing flow.
- Designed the AI expense-analysis flow.

**06/06**
- Divided the work among team members.
- Planned the Backend, Frontend, and AWS deployment work.

**07/06**
- Consolidated the research documentation.
- Prepared for the architecture design phase.

### Results
- Built a deployment approach for the Budget Tracker system on AWS.
- Fully identified all the AWS services to be used.
- Completed the deployment plan.

## Week 6 (25/05 – 31/05)

### Objectives
- Consolidate and systematize the AWS knowledge learned over the previous 5 weeks.
- Review the AWS services practiced through the labs.
- Reinforce the knowledge learned in Module 01 and Module 02.
- Explore how to apply AWS infrastructure to the team's project.

### Activities

**25/05**
- Reviewed and consolidated the AWS services learned so far.
- Studied an overview of the ZeroBug Agent project and its AWS deployment requirements.

**26/05**
- Reviewed identity and access management concepts.
- Consolidated knowledge on IAM User, IAM Group, IAM Role, and the Switch Role mechanism.
- Evaluated the role of IAM in managing system access.

**27/05**
- Reviewed AWS network infrastructure knowledge.
- Went over the process of building a VPC, Public Subnet, Private Subnet, Internet Gateway, NAT Gateway, Route Table, and Security Group.
- Consolidated knowledge on network security and connectivity between system components.

**28/05**
- Reviewed knowledge on deploying and managing servers on AWS.
- Consolidated the process of launching Amazon EC2, configuring an Application Load Balancer, and connecting applications.
- Reviewed application deployment and system scalability concepts.

**29/05**
- Reviewed knowledge on Amazon SQS and other services supporting application deployment.
- Consolidated architecture options suitable for the project's requirements.

**30/05**
- Analyzed the requirements of the ZeroBug Agent project.
- Compared the AWS services learned against the planned architecture.
- Discussed and defined the team's work assignments.

**31/05**
- Consolidated all knowledge learned over the 6 weeks.
- Compared the AWS services learned against the team project's requirements.
- Built an approach for applying AWS infrastructure to the system's deployment architecture and planned research on the remaining services needed.

### Results
- Systematized all the AWS knowledge learned over the previous 5 weeks.
- Reinforced knowledge on IAM, VPC, EC2, Application Load Balancer, and Amazon SQS.
- Clearly understood how AWS services relate to one another within a complete deployment architecture.
- Applied the AWS knowledge learned to build a deployment approach for the team project, laying the foundation for the design and development phase.
